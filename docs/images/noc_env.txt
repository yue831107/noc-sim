================================================================================
  Software / Hardware Architecture
================================================================================

  User Code / RTL Testbench
  ──────────────────────────────────
    - 使用者 C++ test program
    - SystemVerilog testbench
    - Python/shell test scripts

  NocSystem Public API (noc_api.h)
  ──────────────────────────────────────────
    - Group A: Construction & Configuration
    - Group B: Transaction Submission (High-Level)
    - Group C: Simulation Control
    - Group D: Completion & Response
    - Group E: Metrics & Verification
    - Group F: Debug & Introspection + Output Pattern Generation

  Internal Components
  ─────────────────────────────
    - TrafficManager: Transaction lifecycle management
    - Mesh: Topology + wiring via Channel<T>
    - CppRouter: 6-stage pipeline (InputQ/RC/VA/SA/ST/OutputQ)
    - CppNI: NMU (Pack/Unpack/Inject/RoB) + NSU (Unpack/Reassembly/Pack)
    - Channel<T>: Configurable-delay link model
    - InputBuffer / BufferState: Separated data + credit tracking
    - Allocator: Pluggable (RoundRobin / iSLIP / QoSAwareRR)
    - Scoreboard / MetricsCollector
    - HostMemory / LocalMemory

    ┌──────────────── HOT-SWAP BOUNDARY ────────────────┐
    │  CppRouter ←→ RouterDpiBridge (Router_Interface)   │
    │  CppNI     ←→ NIDpiBridge     (NI_Interface)       │
    └────────────────────────────────────────────────────┘

  Co-Sim Bridge + RTL
  ─────────────────────────────
    - DPI-C thin wrapper: noc_init / noc_step / noc_submit_* / noc_get_*
    - RouterDpiBridge / NIDpiBridge
    - SystemVerilog RTL modules

================================================================================
  Data Flow
================================================================================

  Request Path:
    AXI Master → NMU (PackAW/W/AR + ECC) → Req Router chain → NSU (UnpackAW/W/AR + ECC check)
                                                                → Local Memory write/read

  Response Path:
    Local Memory → NSU (PackB/R + ECC) → Rsp Router chain → NMU (UnpackB/R + ECC check)
                                          ↑                   → RoB → Completion
                                          │
                                    (Multicast B: In-Network Reduction at each hop)
